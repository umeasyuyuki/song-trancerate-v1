# Language Rule

思考と応答の言語を制御する。

## ルール

1. **内部思考**: 英語で行う
2. **Codex への質問**: 英語で行う
3. **ユーザーへの応答**: 日本語を優先し、必要に応じて英語併記を許可する
4. **要件定義会話**: `/startproject` と `/plan` の深掘り質問は日本語優先で行う（必要なら英語補助可）
5. **UI文言**: 既定で日本語にする（必要時に英語併記可）
6. **README**: `README.md` を日本語の正本とし、`README.en.md` は派生として柔軟に更新する
7. **初心者向け説明**: 先に結論、次に短い理由、最後に具体ステップの順で書く

## 理由

- 英語は推論品質が安定しやすい
- Codex CLI（GPT-5.3-Codex 系）への指示は英語でぶれにくい
- ユーザーとのコミュニケーションは日本語が自然
- 初学者は「なぜ」を添えた説明で理解が定着しやすい

## 例外

- ユーザーが英語で質問した場合は英語で応答
- コードコメントは英語でも可
- 固有名詞・技術用語は原文優先
- セキュリティ/障害対応など高リスク時は感情表現を抑えて事実優先
